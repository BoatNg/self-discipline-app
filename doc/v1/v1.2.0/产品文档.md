# 月视图展示优化 PRD

## 1. 背景与目标

当前记录页面采用的单日任务状态展示方式已无法满足用户对长期行为管理的需求，尤其是在多任务并行管理时，用户需要一种直观、便捷的方式来快速查看每个月的行为状态。

**本次迭代目标：**

* 将任务状态的展示从单日视图扩展为**月视图**
* 通过颜色、图标等方式，帮助用户一眼识别每一天的任务状态
* 提高用户对任务进度的整体感知，特别是在处理多任务并行时

---

## 2. 页面结构与展示方式

### 2.1 月视图布局

* **月视图**：按月份展示，网格状布局，每行表示一周，每列表示一天。
* 每个日期格中显示该天多项任务的状态信息，支持不同任务并行管理。
* 每个任务的状态以颜色、图标等标识进行展示。

> 每个月显示为一个网格，用户可以在此视图中直观看到每个任务在每一天的完成情况。

### 2.2 任务状态展示

任务状态采用图标与颜色的组合来标识，不同任务的状态显示如下：

| 状态  | 含义     | 颜色/图标 |
| --- | ------ | ----- |
| 成功  | 任务完成   | 🟢    |
| 失败  | 任务未完成  | 🔴    |
| 未记录 | 任务未记录  | ⚪️    |
| 自定义 | 任务类型状态 | 任务图标  |

* 每个任务的标识可以根据任务类型（如禁烟、运动等）变化，使用相应图标（例如禁烟任务可以用🚭，运动任务可以用🏃）。
* 如果一天内所有任务都完成，可以突出显示该天，如绿色框标示。
* 如果当天有失败或未记录的任务，显示红色警示，提醒用户关注。

### 2.3 用户交互

* **点击日期**：用户点击某一日期格后，弹出该天的所有任务详情或进一步的记录。
* **连续查看**：用户可以通过日历快速切换不同日期的任务进度，查看某一任务在不同日期的表现。

---

## 3. 数据结构与实现

### 3.1 数据支持要求

* 每个日期格需要能够容纳多项任务的状态信息，每个任务有对应的状态（成功、失败、未记录等）。
* 后端系统需要传递每个任务在特定日期的状态信息，以便前端展示。
* 任务的状态信息包括任务类型、完成情况（完成、失败、未记录）以及对应的图标。

### 3.2 前端显示需求

* 前端需要支持月视图的网格布局，并能够展示每个任务在每个日期的状态。
* 每个日期格内需要能够显示多项任务状态，通过颜色、图标等直观展示任务的完成情况。

---

## 4. 设计要求与用户体验

### 4.1 视觉设计

* 使用不同颜色和图标进行任务状态区分。
* 当某一天所有任务都完成时，使用突出显示效果（如绿色框）提醒用户。
* 如果一天内有失败或未记录的任务，使用红色提示。
* 每个任务的图标或颜色需要与任务类型一致，确保直观识别。

### 4.2 交互设计
* 可以选择月份，展示该月所有任务状态。


---

## 5. 实施步骤

### 5.1 前端开发

* 实现月视图布局，确保每个日期格可以容纳多个任务状态。
* 为每个任务状态配置图标和颜色，确保任务类型与显示一致。


**本 PRD 为月视图展示优化的唯一规则依据。**
